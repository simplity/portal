3. Application Configuration
  • Create component folder structure under WEB-INF as follows:
      • comp
      • dt
      • msg
      • rec
      • service
        • tp
      • sql
  • Configure application.xml under WEB-INF/comp as follows.
    <?xml version="1.0" encoding="UTF-8"?>
    <application
            xmlns="http://www.simplity.org/schema"
            applicationId="HelloWorld"
            autoLoginUserId="100"
            dbVendor="oracle"
            dbDriverClassName="oracle.jdbc.OracleDriver"
            connectionString="database connection string"
            logSqls="true">
    </application>
      
    • This is the root file of simplity which helps to configure various features of Simplity. Here "applicationId" is a mandatory field which represents the application name. applicationId should be a unique name for the application.
    • NOTE: To get more help, click ctrl + tab from the eclipse editor.
  • Create Simplity record under WEB-INF/comp/rec as follows.
    <?xml version="1.0" encoding="UTF-8" standalone="no"?>
    <record name="Employee"
            recordType="storage"
            tableName="EMPLOYEE"
            keyToBeGenerated="true"
            okToSelectAll="true"
            xmlns="http://www.simplity.org/schema">
      <fields>
        <field columnName="EMPLOYEE_ID" dataType="_number" fieldType="primaryKey" name="employeeId" sqlTypeName="INTEGER"/>
        <field columnName="EMPLOYEE_NAME" dataType="_text" fieldType="data" name="employeeName" sqlTypeName="VARCHAR"/>
      </fields>
    </record>

    • Simplity records are kept under the comp/rec folder. This is the main part of data model. Data structures that are used as input to, or output from a service are modeled as records. Following are the fields of simplity record:
    • name (mandatory field) - name of the record or entity
    • moduleName (optional field) - sub-folder name inside rec folder. In this example(rec/login/user.xml) record name is user.xml and module name login.
    • recordType (mandatory field) - STORAGE (represents a row of a table being stored in an rdbms), VIEW (a view involving one or more storage records), STRUCTURE (record structure used by the application, but is not linked to any stored data).
    • tableName (optional field) - name of the db table, if recordType is STORAGE or VIEW.
    • keyToBeGenerated (optional field) - true/false (if any table column value to be generated automatically)
    • okToSelectAll (optional field) - true/false(set true to select all rows from the table)
  • Create Simplity service under WEB-INF/comp/service/tp as follows.
    <?xml version="1.0" encoding="UTF-8"?>
    <service name="saveEmployee" xmlns="http://www.simplity.org/schema">

        <inputData>
            <inputFields>
                <inputField dataType="_text" name="employeeName"/>
                <inputField dataType="number" name="employeeId"/>
          <inputFields>
        </inputData>

        <actions>
            //perform save action
            <save saveAction="add" recordName="employee">
            //set output status
            <setValue fieldName="dbaction" fieldValue="Record saved successfully"></setValue>
        </actions>

        <outputData fieldNames="dbaction"></outputData>

    </service>

  • Create client(helloworld.html) as follows.
    <!DOCTYPE html>
    <html>
      <head>
        <script type="text/javascript" src="js/simplity.js"></script>
      </head>
      <body>
      <form id="empform">
          <input name="employeeId" type="text" />
          <input name="employeeName" type="text" />
          <button onclick="saveEmployeeRecord()">Save</button>
      </form>
        <script type="text/javascript">
          var saveEmployeeRecord = function() {
            var employeeId = document.getElementById("empform").employeeId.value);
            var employeeName = document.getElementById("empform").employeeName.value);
            var data = "{'employeeId':employeeId,'employeeName':employeeName}";
            Simplity.getResponse("saveEmployee",data,onSuccess);
          }
          var onSuccess = function(json) {
            console.log(json);
          }
        </script>
      </body>
    </html>

Utilities
1. Steps to generate Simplity Records using Database schema
  • Create a new "Maven Project" in eclipse
  • Configure pox.xml to have the following entry
    • Downloads
    • Copy code
    • View output
    <build>
      <plugins>
        <plugin>
          <groupId>org.apache.torque</groupId>
          <artifactId>torque-maven-plugin</artifactId>
          <version>4.0</version>
          <executions>
            <execution>
              <id>generate-schema-from-jdbc</id>
              <phase>generate-test-sources</phase>
              <goals>
                <goal>generate</goal>
              </goals>
              <configuration>
                <packaging>classpath</packaging>
                <configPackage>com.infosys.qreuse.torque.simplity</configPackage>
                <newFileTargetDir>target/generated-schema</newFileTargetDir>
                <newFileTargetDirUsage>none</newFileTargetDirUsage>
                <options>
                  <torque.jdbc2schema.driver>org.h2.Driver</torque.jdbc2schema.driver>
                  <torque.jdbc2schema.url>jdbc:h2://<host>:<port>/<database name></torque.jdbc2schema.url>
                  <torque.jdbc2schema.user>username</torque.jdbc2schema.user>
                  <torque.jdbc2schema.password>password</torque.jdbc2schema.password>
                  <torque.jdbc2schema.schema>schema name</torque.jdbc2schema.schema>
                </options>
              </configuration>
            </execution>
          </executions>
          <dependencies>
            <dependency>
              <groupId>com.h2database</groupId>
              <artifactId>h2</artifactId>
              <version>1.3.171</version>
            </dependency>
            <dependency>
              <groupId>com.infosys.qreuse.torque.simplity</groupId>
              <artifactId>template</artifactId>
              <version>0.0.1</version>
            </dependency>
          </dependencies>
        </plugin>
      </plugins>
    </build>

  • To generate the records use the command "mvn test"
  • Generated records are present under "target/generated-sources" folder.
Creating Simplity application using archetype
1. Get simplity kernel
  • Install the simplity archetype-partial at https://github.com/simplity/simplity/tree/master/utils in your local repo.
  • Create a sample maven web application in eclipse
  • Right click on your application → run as → maven build
  • In Goals provide the archetype info as below
    • Downloads
    • Copy code
    • View output
    -DinteractiveMode=false -DarchetypeCatalog=local -DarchetypeGroupId=<groupid of archetype in your local repo> -DarchetypeArtifactId=<artifactId of archetype in your local repo> -DarchetypeVersion=<version of archetype in your local repo> -DgroupId=<groupId of your project> -DartifactId=<artifactId of your project> -Dpackage=<package name you want to create in your project>

  • Apply and run the configuration to generate the simplity folder structure in your application from archetype.
