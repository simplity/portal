Features
We provide the best solutions
  • Read Action
    Read a row from the underlying table/view based on the primary key value.
    LEARN MORE
  • ReadWithSql Action
    Use a sql to read data rows. Refer to sql definitions as a buildina block for details.
    LEARN MORE
  • Filter Action
    Read a row from the underlying table/view based on the filtering criteria.
    LEARN MORE
  • ReadChildren Action
    Action that reads rows from one or more child records based on the value of a parent key.
    LEARN MORE
  • Save Action
    Add,update or delete rows of data based on specification in a record along with children if needed.
    LEARN MORE
  • ExecuteSql Action
    If a save action is not convenient for you to design your data operation,you may use this.
    LEARN MORE
  • KeyValueList Action
    This action is designed specifically for server to send list of possible values for a drop-down.
    LEARN MORE
  • BatchProcessor Action
    This action allows the rows/items of the file/DB Query/Messages to be processed as batch.
    LEARN MORE
