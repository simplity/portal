RDBMS based actions
Dealing with database is the core of any transaction processing system. We have designed a set of actions to help you in that. They have the following common attributes (in addition to all the common attributes of action).
Now let us understand RDBMS based actions.
  • Read Action
    Read a row from the underlying table/view based on the primary key value.
    Feature details
  • ReadWithSql Action
    Use a sql to read data rows. Refer to sql definitions as a buildina block for details.
    Feature details
  • Filter Action
    Read a row from the underlying table/view based on the filtering criteria.
    Feature details
  • ReadChildren Action
    rdbmsaction that reads rows from one or more child records based on the value of a parent key.
    Feature details
