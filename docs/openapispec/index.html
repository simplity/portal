Open API Specification
Simplity support Swagger. Swagger is the largest framework of API developer tools for the OpenAPI Specification. Swagger specification creates the RESTful contract for your API, detailing all of its resources and operations in a readable format. It describes the entire API including (available endpoints and operations on each endpoint, input/output parameters for each operation, etc.) For mode detail about Swagger, check out the swagger website.

Open API Structure
Open API specification is written JSON format. It starts with swagger version, info, host, basepath, paths (includes operation, parameters, request body, response, input and output models).

Tip Refer Simplity service to define operations and parameters, and Simplity record to define definitions

Input may contain records. A record may be used to get fields or a list/table of rows. Record is used with specific purpose. “get”, “filter”, “save” ,“list” and “suggest” are possible values.

  • “paths” defines the endpoints of the API and operation supported by these endpoints.

    paths section

    Syntax:

    • "paths": {
          "URL": {
              "get": {
                  "tags": [ "" ],            "operationId": "Simplity service name",
                  "summary": "findUser",
                  "description":"find user service",
                  "deprecated": false,
                  "parameters":[],
                  "responses": {
                      "200": {
                          "description": "find user",
                          "schema": {
                              "$ref": "#/definitions/User"
                          }
                      }
                  }
              }
          }
      }

    Attributes:

    • "URL" - URL path (Example: /users)

    • get - http method

    • operationId - simplity service name

    • parameters - list of input parameters

    • responses - includes status code and schema. schema can be referenced inline or using $ref.


  • “definitions” define common data structures used in API and it is referenced in path section using $ref.

    definitions section

    Syntax:

    • "definitions": {
          "schema": {
              "type": "object",
              "properties": {
                  "propertyName": {
                      "type": "string",
                      "format": ""
                  }
              }
          }
      }

    Attributes:

    • schema - schema name (Example: user)

    • properties - list of properties (includes property name, property type and property format).
