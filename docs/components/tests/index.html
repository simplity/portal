Tests
TestRun is the component to run the test-cases. Essentially we define a sequence of test cases that are run in that order .
Tests are organized in one or more files under test/ folder. Attributes of testrun are:

Attribute name

Required?

Description

testName

yes

Unique name within a module.

moduleName

no

Should match the folder structure under which this component is saved. If a.b.c is the published name of this test then module name is a.b, test name is c, and the component is saved as /a/b/c.xml under the root folder meant for tests (/test/).


TestCase

TestCases make up a TestRun.

Attribute name

Required?

Description

testCaseName

yes

unique name given to this test case.

serviceName

yes

service to be tested.

description

no

description for the test case.

inputJson

no

In case you do not want to specify individual field values but provide a ready string that is to be provided as request pay-load to the service.

inputFields

no

represents fields to be provided as input to a service

refer to attributes of InputField

inputItems

no

Specify a qualified attribute to be used to identify a specific item in the JSON

refer to attributes of InputItem

testForFailure

no

number of errors expected. Non-zero implies that we expect this request to fail, and hence output data expectations are irrelevant.

outputJson

no

if you want to specify exactly the fields/sheets expected as in the pay-load format(json)

outputFields

no

represents the fields to be expected in the output of a service

refer to attributes of OutputField

outputLists

no

represents a list/array of items in a json

refer to attributes of OutputList

outputItems

no

items/objects (which in turn contain fields/lists)

refer to attributes of OutputItem

fieldsToBeAddedToContext

no

fields that are used by subsequent test plans

refer to attributes of ContextField

testClassName

no

Use java to inspect the result and assert. fully qualified class name that implements org.simplity.test.Inspector

nbrAssertions

no

how many assertions are you making in this test case? This is meant for reporting purposes


InputField

Represents a field to be provided as input to a service.

Attribute name

Required?

Description

fieldSelector

yes

qualified field name which is relative to its parent.

fieldValue

yes

field value or $variableName to get value from test context


InputItem

Represents a JSON object.Allows setting assertions on an item

Attribute name

Required?

Description

itemSelector

yes

qualified name to get the row data. 0 for first row, a.2 to get third row of the array with attribute a etc..

inputFields

yes

fields specified for this selected item.


OutputField

Represents a field that is expected to be in output of a service.

Attribute name

Required?

Description

fieldSelector

yes

qualified field name which is relative to its parent.

fieldValue

yes

field value or $variableName to get value from test context

shouldBeAbsent

no

do you want test for non existence of this field?


OutputList

Represents a list/array of items in a json.

Attribute name

Required?

Description

listSelector

yes

qualified name that would fetch this list (array) for example a.2.b would get json[a][2][b].

minRows

yes

min rows expected.

maxRows

yes

max rows expected.


OutputItem

Represents a JSON object. It Allows setting assertions on such an item.

Attribute name

Required?

Description

itemSelector

yes

qualified name to get the row data. 0 for first row, a.2 to get third row of the array with attribute a etc..

outputFields

yes

fields specified for this selected row.

refer to attributes of OutputField


ContextField

Specification for a field from output of a service test to be added to the test context.

Attribute name

Required?

Description

fieldSelector

yes

source of this field in the output JSON e.g. customerName or orders.lines[2].price. Special case "." (just dot ) to select the entire JSON itself

nameInContext

yes

name under which this is to be added to the context. This is the name that the next service uses to retrieve this value. defaults to fieldSelector
