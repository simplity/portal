Batch
Batch is a list of all the jobs to be managed as batch/background services. This component allows you to specify all the jobs to be executed. This component is to be configured in application.xml using batchJobToRunOnStartup.

Attribute name

Required?

Description

name

yes

Unique within a module.

moduleName

no

Should match the folder structure under which this component is saved. If a.b.c is the published name of this batch then module name is a.b, batch name is c, and the component is saved as /a/b/c.xml under the root folder meant for batch jobs (/batch/).

defaultUserId

no

userId to be used for running the batch. Defaults to setting in application.xml, or "100". Over-ridden by the setting at job level.


Elements available for Batch are : Jobs


Following are the job attributes

Attribute name

Required?

Description

name

yes

name of the job, unique within a jobs collection

serviceName

yes

qualified name of the service to be run as a job

runAtTheseTimes

no

If the scheduling is by time of the day, use this option to specify comma separated list of times in 24Hr format upto minute (hh:mm, hh:mm, …​)

runInterval

no

Use this if batch service to be run repeatedly. number of seconds between two runs start-to-start.

nbrDedicatedThreads

no

use this if this is a listener/background job that is run once. min value is 1. If more than 1 is specified, the job is fired on that many threads simultneously, subject to availablity of threads in this scheduler.

userId

no

userId to be used for running this job. Overrides setting at jobs/application. Service is executed with thisuser as logged-in user

inputFields

no

refer to attributes of Input Field

inputJson

no

parameters that the service expects as input are supplied with this mechanism
