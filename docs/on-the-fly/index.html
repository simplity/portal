On-the-fly Services
Expected input for a on-the-fly service can be specified. Simplity parses request data for these input and invokes the requested on-the-fly service only if the input data passes all naming convention and validations. In case of any validation error, a response is sent back to client with appropriate messages.In case of success, on-the-fly service fetches the data from database depending upon the input given and sends it back to client.

Tip On-the-fly services uses only records and input data.

  1. Input may contain records. A record may be used to get fields or a list/table of rows.

    1. Record is used with specific purpose. “get”, “filter”, “save” ,“list” and “suggest” are possible values.

      • “get” means it is meant to get this record with primary key. Only the primary key field of the record is expected as input. Other fields are all ignored. Check out the helloworld example: helloworld.

        Note

        Examples:

        • For record agents , Primary key is agentCode.If user wants to get the details about 'A004' agentCode then while calling the function user has to provide the below details :

          \nvar data = "{'agentCode':'A004'}";
 Simplity.getResponse("get_tables.agents",data,onSuccess,onError);

    2. Composite Primary key : For record compParent , Primary key is parentKeyText and parentKeyNbr.If user wants to get the details about 'k1' parentKeyText and '1' parentKeyNbr then while calling the function user has to provide the below details :

      \nvar data = "{'parentKeyText':'k1','parentKeyNbr':'1'}";
Simplity.getResponse("get_tables.compParent",data,onSuccess,onError);

    3. filter means the fields in this record are used to specify filtering criterion to get a subset of rows .

      • Each field is assumed to be optional.

      • Field value is validated for the value-type (like numeric/date etc.) but dataType specification is not used for validation. For example a dataType may have min-length of 5, but as filter field we accept even if only one character is specified.

      • Each field may optionally have an associated condition named as fieldNameOperator. For example if fieldName=”cusomerName” then customerNameOperator. If this is missing, a value of ‘=’ is assumed.

        • ‘=’ means exact match.

        • ‘!=’ means should not match.

        • ‘<’ means less than. Valid only for numeric and date fields.

        • ‘>’ means greater than. Valid for numeric and date fields.

        • ‘⇐’ means less than or equal. Valid for numeric and date fields.

        • ‘>=’ means greater than or equal. Valid for numeric and date fields.

        • ‘><’ means value is to be a range(inclusive). fieldValue is from-value. fieldNameTo is the name field. For example amount=”10” amountOperator=”><” amountTo=”20” will match rows with amount field between 10 and 20 (inclusive)

        • ‘~’ means containing this text anywhere

        • ‘^’ means field that starts with this text

        • ‘@’ means matching any one value in the list. fieldValue should have a comma separated list of values. For example color=”red,blue,green”, colorOperator=”@” means rows with color set to red, blue or green.
          Check out the helloworld example: helloworld.

          Note

          Examples :

          • For record agents, if the user wants to filter 'commission' column field from 0.13 to 0.16 then while calling the function user has to provide the below details :

            \nvar data = "{'fieldName':'commission','commission':0.13,'commissionOperator':'><','commissionTo':0.16}";
 Simplity.getResponse("filter_tables.agents",data,onSuccess,onError);

          • Composite Primary key :For record compParent, if the user wants to filter 'createdUser' column field from 100 to 120 then while calling the function user has to provide the below details :

            \nvar data = "{'fieldName':'createdUser','createdUser':'100','createdUserOperator':'><','createdUserTo':'120'}";
Simplity.getResponse("filter_tables.compParent",data,onSuccess,onError);

  • “save” means this is used for add/modify operation.

    • “save” means the row is added if primary key is missing, else is used to modify the row with the specified primary key. Record is updated if primary key values is found, or inserted as a new row if primary key is missing.
      Check out the helloworld example: helloworld.

Note

Examples :

  • For record agents, If the user wants to update the agentName to 'Rosy' depending upon agentCode(primary key) then while calling the function user has to provide the below details :

    \nvar data = "{'agentCode':'A004','agentName':'Rosy'}";
Simplity.getResponse("save_tables.agents",data,onSuccess,onError);

  • Composite Primary key :For record compParent, If the user wants to update the createdUser to '100' and 'text' to 'Dummytext' depending upon primary key then while calling the function user has to provide the below details :

    \nvar data = "{'parentKeyText':'k1','parentKeyNbr':'1','createdUser':'110','text':'Dummytext'}";
 Simplity.getResponse("save_tables.compParent",data,onSuccess,onError);

  • “suggest” means this is used for providing suggested values for a google-suggest-like control on the client.

    • Columns from matching rows are displayed, based on a record definition.

    • Note: For Suggest action, In the Record define suggestionKeyName and suggestionOutputNames as Column field that is required for getting the suggested values and displaying the field values respectively.
      Check out the helloworld example: helloworld.

Note

Examples :

  • For record agents: '_key' is for field value to be searched with suggestionKeyName = "agentCode" and '_matchStarting' is for extracting rows starting with supplied chars. _matchStaring will take boolean values: true and false.

    \nvar data = "{'_key':'A','_matchStarting':true}";

  • Composite Primary key: For record compParent: '_key' is for field value to be searched for with suggestionKeyName="parentKeyNbr".

    \nvar data = "{'_key':'k1'}";
Simplity.getResponse("suggest_tables.compParent",data,onSuccess,onError);

  • “list” is designed specifically for server to send list of possible values for a drop-down for a control (input field) on the client.

    • Record must specify listFieldName to the column that is to be displayed by the client, while the primary key would be the internal value.

    • In case the record has list of values grouped by another key, like states for a given country, then the record must also specify listGroupKeyName. And the client has to send the value for this group field with that field name.
      Check out the helloworld example: helloworld.

Note

Examples :

  • For record agents: '_key' is for field value with listFieldName="agentCode".

    \nvar data = "{'_key':'A'}"; Simplity.getResponse("list_tables.agents",data,onSuccess,onError);

  • Composite Primary key :For record compParent: '_key' is for field value with listFieldName="text".

    \nvar data = "{'_key':'Text'}";
Simplity.getResponse("list_tables.compParent",data,onSuccess,onError);
