Simplity with Batch
You can get started with simplity in various ways.
Steps to implement batch in Simplity
1. Get simplity kernel
For Maven users-
<dependency>
<groupId>org.simplity<groupId>
<artifactIdId>simplity</artifactId>
<version>1.2</version>
</dependency>

Plain Java-
<dependency>
<groupId>org.simplity</groupId>
<artifactIdId>simplity</artifactId>
<version>1.2</version>
</dependency>
2. Creating project
  • Create a new "Maven Project" in eclipse
  • Configure web.xml to have the following entry
    <servlet>
      <servlet-name>SimplityStartup</servlet-name>
      <servlet-class>org.simplity.http.Startup</servlet-class>
      <load-on-startup>1</load-on-startup>
    </servlet>
    <servlet>
      <servlet-name>s</servlet-name>
      <servlet-class>org.simplity.http.Serve</servlet-class>
    </servlet>
    <servlet-mapping>
      <servlet-name>s</servlet-name>
      <url-pattern>*._s</url-pattern>
    </servlet-mapping>

3. Application Configuration
  • Create component folder structure under WEB-INF as follows:
    • comp
      • batch
      • dt
      • msg
      • rec
      • service
        • tp
      • sql

  • Configure application.xml under comp as follows.
    <?xml version="1.0" encoding="UTF-8"?>
    <application
            xmlns="http://www.simplity.org/schema"
            applicationId="HelloWorld"
            autoLoginUserId="100"
            jobsToRunOnStartup="batch"
            logSqls="true">
    </application>
      
    • This is the root file of simplity which helps to configure various features of Simplity. Here "applicationId" is a mandatory field which represents the application name. applicationId should be a unique name for the application.
    • NOTE: To get more help, click ctrl + tab from the eclipse editor.

  • Create Simplity batch under comp/batch as follows.
    <?xml version="1.0" encoding="UTF-8" standalone="no"?>
    <batch name="exampleBatch" xmlns="http://www.simplity.org/schema" defaultUserId="420">
        <jobs>
            <job name="listener" serviceName="listenerService" nbrDedicatedThreads="1">
                <inputFields>
                    <inputField name="maxLoops" value="10" valueType="integer" />
                </inputFields>
            </job>
            <job name="interval" serviceName="intervalService" runInterval="6">
                <inputJson>
                    <![CDATA[{"f1" : true,"f2" : 128,"f3" : "2016-11-23"}]]>
                </inputJson>
            </job>
            <job name="periodic" serviceName="periodicService" runAtTheseTimes="09:19, 09:52, 20:31, 20:34"></job>
        </jobs>
    </batch>

    • nbrDedicatedThreads - indicates the number of threads to run this job (based on the availability of threads).
    • runInterval - job runs at every given seconds.
    • runAtTheseTimes - job runs at given time(times-of-day).

  • Example of Interval Batch
    • Create Simplity service under comp/service/tp as follows.
    • runInterval - used to implement the periodic batch
      <?xml version="1.0" encoding="UTF-8" standalone="no"?>
          <service
          xmlns="http://www.simplity.org/schema"
          name="intervalService"
          dbAccessType="none"
          description="This service is fired every so-many seconds by the batch manager.
          But we can also use this service in any other context"
          >
          <actions>
              <setValue
                  fieldName="log"
                  fieldValue="This is interval service starting its work" />
              <log names="log" />
              <logic
                  description="this logic is supplied as an example in core library sleeps for upto 10 seconds before saying hello"
                  className="org.simplity.ide.AsynchHelloWorld" />
              <setValue
                  fieldName="log"
                  fieldValue="Interval service completed its work" />
              <log names="log" />
          </actions>
      </service>

    • This job runs at every 6 seconds

  • Example of Periodic Batch
    • Create Simplity service under comp/service/tp as follows.
    • runAtTheseTimes - used to implement the interval batch
      <?xml version="1.0" encoding="UTF-8" standalone="no"?>
          <service
          xmlns="http://www.simplity.org/schema"
          name="periodicService"
          dbAccessType="none"
          description="This service is fired periodically, that is, at certain time-of-day"
          >
          <actions>
              <setValue
                  fieldName="log"
                  fieldValue="This is PERIODIC service cranking-up" />
              <log names="log" />
              <logic
                  description="this logic is supplied as an example in core library sleeps for upto 10 seconds before saying hello"
                  className="org.simplity.ide.AsynchHelloWorld" />
              <setValue
                  fieldName="log"
                  fieldValue="PERIODIC service is done with" />
              <log names="log" />
          </actions>
      </service>

    • This job runs at specified time of the day

  • Create client(batchExample.html) as follows.
    <!DOCTYPE html>
    <html>
        <head>
            <script type="text/javascript" src="js/simplity.js"></script>
        </head>
        <body>
            <div id="response"></div>
            <br>
            <br>
            <button onClick="getBatchStatus()">Get batch status</button>
            <br>
            <br>
            <button onClick="startBatch()">Start batch status</button>
            <br>
            <br>
            <button onClick="stopBatch()">Stop batch status</button>
            <br>
        <script type="text/javascript">
            function onSuccess(response) {
                document.getElementById("response").innerHTML = JSON.stringify(response);
            }
            function onError(response) {
                document.getElementById("response").innerHTML = JSON.stringify(response);
            }
            function getBatchStatus() {
                var data = "{'jobsAction':'status','jobsName':'exampleBatch','jobName':'periodicService'}";
                Simplity.getResponse("batch.exampleBatch", data, onSuccess, onError);
            }
            function startBatch() {
                var data = "{'jobsAction':'start','jobsName':'exampleBatch','jobName':'periodicService'}";
                Simplity.getResponse("batch.exampleBatch", data, onSuccess, onError);
            }
            function stopBatch() {
                var data = "{'jobsAction':'stop','jobsName':'exampleBatch','jobName':'periodicService'}";
                Simplity.getResponse("batch.exampleBatch", data, onSuccess, onError);
            }
        </script>
        </body>
    </html>
