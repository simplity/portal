Simplity with Grizzly Server
Run Simplity application using Grizzly server.
Steps to create Simplity application with Spring Boot
1. Get simplity kernel
For Maven users-
<dependency>
    <groupId>org.simplity<groupId>
    <artifactIdId>simplity</artifactId>
    <version>1.2</version>
</dependency>

Plain Java-
<dependency>
    <groupId>org.simplity</groupId>
    <artifactIdId>simplity</artifactId>
    <version>1.2</version>
</dependency>
2. Get grizzly dependency
<dependency>
    <groupId>org.glassfish.jersey.containers<groupId>
    <artifactIdId>jersey-container-grizzly2-http</artifactId>
    <version>2.25.1</version>
</dependency>

<dependency>
    <groupId>org.glassfish.jersey.containers</groupId>
    <artifactId>jersey-container-grizzly2-servlet</artifactId>
    <version>2.25.1</version>
</dependency>

<dependency>
    <groupId>org.glassfish.jersey.media</groupId>
    <artifactId>jersey-media-json-processing</artifactId>
    <version>2.25.1</version>
</dependency>

2. Creating project and Configuration
  • Create a new "Maven Project" in eclipse
  • Create component folder structure under resources as follows:
    • comp
      • dt
      • msg
      • openapi
      • rec
      • service
        • tp
      • sql


  • Configure application.xml under resources/comp as follows.
    <?xml version="1.0" encoding="UTF-8"?>
    <application
            xmlns="http://www.simplity.org/schema"
            applicationId="ContractService"
            autoLoginUserId="100"
            logSqls="true">
    </application>
      
    • This is the root file of simplity which helps to configure various features of Simplity. Here "applicationId" is a mandatory field which represents the application name. applicationId should be a unique name for the application.
    • NOTE: To get more help, click ctrl + tab from the eclipse editor.

  • Create Open API under resources/comp/openapi as follows.

  • Create Simplity record under resources/comp/rec as follows.
    <?xml version="1.0" encoding="UTF-8" standalone="no"?>
    <record name="ContractHeader"
            recordType="storage"
            tableName="CONTRACTHEADER"
            keyToBeGenerated="false"
            okToSelectAll="true"
            xmlns="http://www.simplity.org/schema">
      <fields>
        <field columnName="BP_CONTRACT_NUM" dataType="_text" fieldType="primaryKey" name="bpContractNum"         sqlTypeName="VARCHAR"/>
        <field columnName="DESCRIPTION" dataType="_text" fieldType="data" name="description" sqlTypeName="VARCHAR"/>
        <field columnName="ASSET_NAME" dataType="_text" fieldType="data" name="assetName" sqlTypeName="VARCHAR"/>
      </fields>
    </record>

    • Simplity records are kept under the comp/rec folder. This is the main part of data model. Data structures that are used as input to, or output from a service are modeled as records. Following are the fields of simplity record:
    • name (mandatory field) - name of the record or entity
    • moduleName (optional field) - sub-folder name inside rec folder. In this example(rec/login/user.xml) record name is user.xml and module name login.
    • recordType (mandatory field) - STORAGE (represents a row of a table being stored in an rdbms), VIEW (a view involving one or more storage records), STRUCTURE (record structure used by the application, but is not linked to any stored data).
    • tableName (optional field) - name of the db table, if recordType is STORAGE or VIEW.
    • keyToBeGenerated (optional field) - true/false (if any table column value to be generated automatically)
    • okToSelectAll (optional field) - true/false(set true to select all rows from the table)

  • Create Simplity service under resources/comp/service/tp as follows.
    <?xml version="1.0" encoding="UTF-8" standalone="no"?>
    <service xmlns="http://www.simplity.org/schema"
            name="contractFind"
            dbAccessType="readOnly">
        <actions>
            <filter filterRecordName="filteredContractHeaders" outputSheetName="contractHeaders">
            </filter>
            <log names="contractHeaders"/>
        </actions>
        <outputData>
            <outputRecords>
                <outputRecord sheetName="contractHeaders" />
            </outputRecords>
        </outputData>
    </service>

3. Write Java main program
  • Write Java program and configure Grizzly server,

    import java.io.File;
    import java.net.URI;

    import javax.servlet.ServletRegistration;

    import org.glassfish.grizzly.http.server.CLStaticHttpHandler;
    import org.glassfish.grizzly.http.server.HttpServer;
    import org.glassfish.grizzly.servlet.WebappContext;
    import org.glassfish.jersey.grizzly2.httpserver.GrizzlyHttpServerFactory;
    import org.simplity.kernel.Application;
    import org.simplity.rest.Operations;
    import org.slf4j.Logger;
    import org.slf4j.LoggerFactory;

    public class App {
        public static HttpServer server;
        final static Logger logger = LoggerFactory.getLogger(App.class);

        public static void main(String[] args) {
            try {
                File jarPath = new File(App.class.getProtectionDomain().getCodeSource().getLocation().getPath());
                String folder = jarPath.getParent() + File.separator + "classes/comp" + File.separator;

                try {
                    Application.bootStrap(folder);
                    Operations.loadAll(folder + "/openapi/");
                } catch (Exception e) {
                    logger.error("error while bootstrapping with compFolder=" + folder, e);
                    return;
                }

                WebappContext wContext = new WebappContext("ContractService Context");
                ServletRegistration rRegistration = wContext.addServlet("RestSimplity", org.simplity.rest.Serve.class);
                rRegistration.addMapping("/app/*");

                server = GrizzlyHttpServerFactory.createHttpServer(new URI("http://localhost:8070"));
                wContext.deploy(server);
                server.getServerConfiguration().addHttpHandler(
                        new CLStaticHttpHandler(HttpServer.class.getClassLoader(), "./webapp/"), "/webapp/*");
                server.start();
                Thread.currentThread().join();

            } catch (Exception e) {
                logger.error("Error in Service", e);
                server.shutdown();
            }
        }

    }

3. Run the application
  • In eclipse, right-click on the project --> Run As --> Java Application (Select App.java as Main program)
  • Enter the below URL from the web browser,
    "http://localhost:8070/ContractService/app/openapi/contract/"

    • Note :
      • ContractService - application name.
      • /app/openapi - basePath.
      • /contract - path to the particular service.
